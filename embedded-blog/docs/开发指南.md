## 分层设计

### 1. 应用层 (`src/app`)

**职责**: 应用壳、hash 路由、模块注册中心

#### bootstrap.ts

应用启动入口，负责：
- 初始化应用
- 注册模块
- 绑定全局事件
- 设置主题

#### router.ts

路由管理，负责：
- 解析 hash 路由
- 触发路由变化事件
- 管理路由历史

#### moduleRegistry.ts

模块注册中心，负责：
- 注册页面模块
- 管理模块列表
- 提供模块查询接口

### 2. 功能层 (`src/features`)

**职责**: 业务模块实现

每个功能模块包含：
- `view.ts` - 视图渲染
- `logic.ts` - 业务逻辑（可选）

#### 模块列表

- **home** - 首页（Hero + 导航舞台 + 粒子特效）
- **docs** - 技术文档（分类筛选 + 标签系统）
- **projects** - 项目作品（卡片展示 + 模态框）
- **life** - 个人分享（背景轮播 + 时间线）
- **board** - 留言板（本地存储 + 实时更新）
- **playground** - 趣味实验室（小游戏 + 彩蛋）
- **pet** - 宠物系统（桌面宠物 + 交互）

### 3. 内容层 (`src/content`)

**职责**: 内容数据管理

#### schema.ts

使用 Zod 定义数据结构：
- `projectSchema` - 项目作品验证
- `docSchema` - 技术文档验证
- `lifePostSchema` - 个人分享验证

#### 数据文件

- `projects.ts` - 项目作品列表
- `docs.ts` - 技术文档列表
- `lifePosts.ts` - 个人分享列表

### 4. 共享层 (`src/shared`)

**职责**: 通用工具和动画

#### motion.ts

动画工具库：
- `animateViewEnter(container)` - 页面入场动画
- `bindHoverLift(selector)` - 悬浮提升效果
- `animateSwapOutIn(target, beforeIn)` - 内容切换动画

### 5. 状态层 (`src/store`)

**职责**: 游戏积分和成绩状态管理

## 模块开发

### 新增功能模块

#### 步骤 1: 创建模块文件夹

```bash
mkdir src/features/my-feature
touch src/features/my-feature/view.ts
```

#### 步骤 2: 实现视图渲染

```typescript
// src/features/my-feature/view.ts

export function renderMyFeature(): string {
  return `
    <div class="page-wrapper my-feature-page">
      <section class="container section">
        <h1>我的新功能</h1>
        <p>这里是内容...</p>
      </section>
    </div>
  `;
}

export function bindMyFeature(): void {
  // 绑定事件和交互逻辑
}
```

#### 步骤 3: 注册模块

在 `src/app/bootstrap.ts` 中：

```typescript
import { renderMyFeature, bindMyFeature } from "../features/my-feature/view";

function registerDefaults(): void {
  // ... 现有注册
  
  register({
    key: "my-feature",
    label: "我的功能",
    title: "我的功能",
    visibleInNav: true,  // 是否在导航栏显示
    render: renderMyFeature,
    afterMount: bindMyFeature
  });
}
```

#### 步骤 4: 添加样式

在 `src/style.css` 末尾添加：

```css
/* My Feature Styles */
.my-feature-page {
  /* 页面样式 */
}

.my-feature-page::before {
  background-image: var(--bg-my-feature);
}
```

### 模块开发规范

#### 1. 视图函数

- 函数名以 `render` 开头
- 返回 HTML 字符串
- 使用模板字符串
- 保持结构清晰

#### 2. 逻辑函数

- 函数名以 `bind` 开头
- 负责事件绑定和交互
- 在 `afterMount` 中调用
- 避免全局污染

#### 3. 样式规范

- 使用 CSS 变量
- 类名使用 kebab-case
- 避免硬编码颜色
- 遵循 BEM 命名（可选）

## 开发规范

### 代码约束

#### TypeScript

- 使用类型注解
- 避免 `any` 类型
- 使用 Zod 进行数据验证
- 遵循 ESLint 规则

#### CSS

- 使用 CSS 变量
- 优先使用 flex/grid 布局
- 避免硬编码魔法值
- 样式保持 token 化

#### Python

- 遵循 PEP 8
- 添加错误处理
- 使用 UTF-8 编码
- 添加注释说明

### 数据验证

所有数据必须通过 `src/content/schema.ts` 校验：

```typescript
import { projectSchema } from "./schema";

const project = projectSchema.parse({
  id: "my-project",
  title: "我的项目",
  // ...
});
```

### 模块隔离

- Feature 不得直接依赖其他 Feature 的内部实现
- 通过 moduleRegistry 进行模块间通信
- 保持模块独立性

### 彩蛋功能规范

- 默认低干扰
- 可关闭
- 不抢占主阅读路径
- 入口视觉保持克制

## 部署指南

### 本地测试

```bash
# 启动开发服务器
npm run dev

# 访问 http://localhost:5173/embedded-blog/
```

### 构建生产版本

```bash
# 构建
npm run build

# 预览
npm run preview
```

### 部署到 Gitee Pages

```bash
# 构建
npm run build

# 部署
npm run deploy
```

**注意**：
- 确保 `vite.config.ts` 中的 `base` 配置正确
- 当前配置：`base: '/embedded-blog/'`
- 对应 Gitee Pages 地址：`https://www223.gitee.io/embedded-blog/`

### 环境变量

项目使用 Vite 的环境变量：

- `import.meta.env.BASE_URL` - 基础路径（从 vite.config.ts 读取）
- `import.meta.env.DEV` - 是否为开发环境
- `import.meta.env.PROD` - 是否为生产环境

**使用示例**：

```typescript
const base = import.meta.env.BASE_URL;
const imageUrl = `${base}my-image.jpg`;
```

## Bug排查

### 常见问题分类

#### 1. 编码问题

**症状**：
- 中文显示乱码
- 终端日志显示 `鉃?` 而不是 `✓`
- 文件内容在 IDE 中正常，但在浏览器中乱码

**原因**：
- Windows 默认使用 GBK 编码
- Web 工具链默认使用 UTF-8

**解决方案**：
1. 确保所有文件使用 UTF-8 编码保存
2. VS Code 设置：右下角编码 → 选择 "UTF-8"
3. 参考 [BugList](./BugList.md) 中的详细解决方案

#### 2. 路径问题

**症状**：
- 图片无法加载（404）
- 背景图片显示为黑色
- 资源加载失败

**原因**：
- 使用了绝对路径 `/image.jpg`
- 没有考虑 `base` 路径配置

**解决方案**：
1. 使用 `import.meta.env.BASE_URL` 拼接路径
2. 示例：`` background-image: url('${base}image.jpg') ``
3. 参考 [BugList](./BugList.md) 中的背景黑屏问题

#### 3. 样式问题

**症状**：
- 样式不生效
- 布局错乱
- 动画卡顿

**排查步骤**：
1. 浏览器开发者工具检查元素
2. 查看 CSS 变量是否正确应用
3. 检查是否有样式冲突
4. 测试响应式断点

#### 4. 内容生成问题

**症状**：
- 新内容不显示
- YAML 解析错误
- 类型验证失败

**排查步骤**：
1. 检查 YAML 文件缩进（用空格不用 Tab）
2. 运行脚本查看错误信息
3. 检查生成的 TypeScript 文件
4. 验证数据结构是否符合 schema

#### 5. 动画问题

**症状**：
- 动画不流畅
- GSAP target not found
- 动画卡住

**排查步骤**：
1. 检查元素是否存在
2. 确认动画时机（DOM 是否已渲染）
3. 查看控制台错误
4. 调整动画参数
