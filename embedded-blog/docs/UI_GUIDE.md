# UI 设计指南

## 设计理念

- 主站表达“专业与可信”，趣味层表达“亲和与记忆点”。
- 先信息层次，再视觉装饰；避免纯炫技动效。
- 首页采用“首屏身份表达 + 下滑导航舞台”双层结构。

## 色彩策略

- 主色：`--accent`（科技蓝）用于导航激活与主操作。
- 中性色：`--bg-*`、`--surface*`、`--line` 控制层次与留白。
- 趣味色：`--play-accent` 仅用于小游戏与彩蛋入口，不扩散到正文区。

## 排版策略

- 标题层级清晰，正文保持中等行高，降低阅读疲劳。
- 卡片摘要保持短句，避免一屏信息过密。
- 标签胶囊统一视觉，减少模块割裂感。

## 主题切换

- 支持明暗模式切换，通过导航栏的主题切换按钮（☀️/🌙）触发
- 切换时所有页面背景会同步进行交叉融合的淡入淡出动画（1.2秒）
- 背景图片会根据主题自动切换：
  - 项目作品：Light模式使用亮色背景图，Dark模式使用暗色背景图
  - 技术文档：Light模式使用亮色背景图，Dark模式使用暗色背景图
  - 留言板：Light模式使用亮色背景图，Dark模式使用太空背景图

## 背景图片管理

- 使用 `bg-slider` 结构实现背景图片的淡入淡出切换
- 每个页面包含两个背景层：`.bg-slide-light` 和 `.bg-slide-dark`
- 背景图片采用 `background-size: cover` 确保适配不同屏幕尺寸
- 项目作品页面背景图片使用 `background-position: center 38%` 优化视觉效果

## 宠物与彩蛋原则

- 宠物是“嵌入式 UI 元素”，应绑定在页面内容区域而非悬浮遮挡。
- 彩蛋入口是惊喜，不是主路径，入口视觉保持克制。
