## 快速开始

### 我是内容管理者

1. 阅读 [用户指南](./用户指南.md) 了解基本操作
2. 使用 `npm run add-card` 快速添加内容
3. 遇到问题查看 [BugList](./BugList.md)

### 我是开发者

1. 阅读 [开发指南](./开发指南.md) 了解项目架构
2. 查看技术栈和分层设计
3. 遵循开发规范进行开发
4. 遇到问题查看 Bug 排查章节

### 我是设计师

1. 阅读 [设计规范](./设计规范.md) 了解设计风格
2. 查看色彩系统和排版规范
3. 遵循动画规范设计交互
4. 保持 UI 一致性

## 文档更新规范

### 更新原则

1. **代码修改后必须更新文档**
   - 新增功能时更新开发指南
   - 修改 UI 时更新设计规范
   - 修复 Bug 时更新 BugList

2. **保持文档同步**
   - 文档与代码保持一致
   - 及时更新过时内容

3. **记录更新信息**
   - 在文档头部记录更新时间
   - 说明更新内容

### 更新流程

1. **确定更新范围**
   - 哪些文档需要更新
   - 更新哪些内容

2. **更新文档**
   - 修改相关章节
   - 更新最后更新时间

3. **审查验证**
   - 检查内容准确性
   - 确保格式正确
