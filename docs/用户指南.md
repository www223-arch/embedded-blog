## 快速开始

### 第一步：启动网站

1. 打开项目文件夹
2. 打开命令行（终端）
3. 运行以下命令：
   ```bash
   npm run dev
   ```
4. 在浏览器中访问显示的地址（通常是 `http://localhost:5173/embedded-blog/`）

### 第二步：了解内容结构

网站包含四种内容类型：

- **技术文档** - 技术文章和教程
- **项目作品** - 你的项目展示
- **个人分享** - 生活、摄影、随笔
- **留言板** - 访客留言

## 添加内容

### 方法一：使用自动化脚本（推荐）

这是最简单的方法，适合所有用户。

#### 添加技术文档

```bash
npm run add-card docs
```

#### 添加项目作品

```bash
npm run add-card projects
```

#### 添加个人分享

```bash
npm run add-card life
```

**操作流程**：

1. 运行对应的命令
2. 系统自动创建卡片模板
3. 编辑卡片内容（见下一节）
4. 运行 `npm run generate-content` 生成内容
5. 刷新浏览器查看效果

### 方法二：使用 YAML 文件（高级）

适合需要批量管理内容的用户。

#### 1. 准备图片

**步骤**：

1. 将图片复制到 `public` 文件夹
2. 文件名使用英文小写加连字符，如 `my-photo.jpg`
3. 避免使用中文和空格

**示例**：

- ✅ 好的文件名：`circuit-board.png`
- ❌ 不好的文件名：`我的电路板 照片.png`

#### 2. 创建 YAML 文件

**添加项目作品**：

在 `content/projects/` 文件夹创建 `my-project.yaml`：

```yaml
id: my-project
title: "我的项目"
summary: "项目简介，一两句话"
stack:
  - "技术1"
  - "技术2"
highlights:
  - "亮点1"
  - "亮点2"
gallery:
  - "/my-photo.jpg"
links:
  - label: "查看详情"
    href: "#"
```

**添加技术文档**：

在 `content/docs/` 文件夹创建 `my-doc.yaml`：

```yaml
id: my-doc
title: "我的文档"
category: "编程语言"
tags:
  - "C++"
  - "教程"
level: "beginner"
updatedAt: "2026-04-04"
summary: "文档简介"
markdown: |
  ## 标题
  
  文档内容...
```

**添加个人分享**：

在 `content/life/` 文件夹创建 `my-post.yaml`：

```yaml
id: my-post
title: "我的分享"
date: "2026-04-04"
tag: "摄影"
summary: "分享简介"
cover: "/my-photo.jpg"
```

#### 3. 生成内容

**重要：每次修改后都要运行！**

```bash
npm run generate-content
```

## 管理图片

### 图片规范

- **支持格式**: .png、.jpg、.jpeg、.gif
- **存放位置**: `public` 文件夹
- **命名规范**: 英文小写 + 连字符，如 `my-photo.jpg`
- **引用方式**: `/文件名.jpg`

### 图片使用示例

1. 放置图片：`public/my-project.jpg`
2. 在 YAML 中引用：`cover: "/my-project.jpg"`

## 编辑和删除内容

### 编辑内容

#### 方法一：编辑 YAML 文件

1. 打开对应的 YAML 文件
   - 项目作品：`content/projects/`
   - 技术文档：`content/docs/`
   - 个人分享：`content/life/`
2. 修改内容
3. 保存文件
4. 运行 `npm run generate-content`
5. 刷新浏览器

#### 方法二：编辑 TypeScript 文件

1. 打开对应的 TypeScript 文件
   - 项目作品：`src/content/projects.ts`
   - 技术文档：`src/content/docs.ts`
   - 个人分享：`src/content/lifePosts.ts`
2. 找到对应的卡片对象
3. 修改内容
4. 保存文件
5. 刷新浏览器

### 删除内容

1. 删除对应的 YAML 文件
2. 或者删除 TypeScript 文件中的卡片对象
3. 运行 `npm run generate-content`
4. 刷新浏览器

## 常见问题

### Q: 添加内容后看不到？

**A**:

1. 确保运行了 `npm run generate-content`
2. 刷新浏览器
3. 检查 YAML 文件格式是否正确

### Q: 图片显示不出来？

**A**:

1. 检查图片是否在 `public` 文件夹
2. 检查图片路径是否正确（以 `/` 开头）
3. 检查文件名是否正确

### Q: YAML 文件格式错误？

**A**:

1. 使用空格缩进，不要用 Tab
2. 确保冒号后有空格
3. 字符串用引号包裹
4. 参考 `content/` 文件夹中的示例文件

### Q: 命令运行失败？

**A**:

1. 确保在项目根目录运行
2. 检查命令拼写是否正确
3. 查看错误信息

### Q: 中文显示乱码？

**A**:

1. 确保文件使用 UTF-8 编码保存
2. 在 VS Code 中，点击右下角编码，选择 "UTF-8"
3. 参考 [BugList](./BugList.md) 中的编码问题解决方案
